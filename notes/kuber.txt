포트 번호의 정체

6443번 포트: Kubernetes API Server
역할: 쿠버네티스 클러스터의 **'입과 귀'**입니다.

누가 쓰나: 모든 워커 노드들이 이 포트를 통해 마스터 노드에게 보고를 합니다. 우리가 터미널에서 kubectl 명령어를 칠 때도 내부적으로는 이 6443 포트를 통해 마스터에게 명령을 전달합니다.

필요한 이유: 이 포트가 막히면 워커 노드가 마스터에게 "나 살아있어!"라고 신호를 못 보내서 NotReady 상태가 됩니다.

10250번 포트: Kubelet API
역할: 각 노드(마스터 포함 모든 노드)에서 돌아가는 관리 요원인 **'Kubelet'**의 포트입니다.

누가 쓰나: 마스터 노드가 워커 노드에게 "컨테이너 상태 좀 보여줘" 혹은 "로그 좀 가져와봐"라고 명령을 내릴 때 이 포트로 접속합니다.

필요한 이유: 이 포트가 막히면 마스터 노드에서 kubectl logs나 kubectl exec 같은 명령어가 작동하지 않습니다.
